**Step A — Verify Vite build works and produces dist/ directory:**

Run `npm run build` in the project root. This runs `vite build` which outputs to `dist/` by default. Verify `dist/index.html` exists after build.

**Step B — Update server.js to serve static files:**

Add these imports at the top of server.js:
```javascript
import { fileURLToPath } from 'node:url';
import { dirname, join } from 'node:path';
```

Add `__dirname` computation after imports:
```javascript
const __dirname = dirname(fileURLToPath(import.meta.url));
```

Add static file serving AFTER `app.use(express.json())` but BEFORE API route mounts:
```javascript
// Serve built frontend static files
app.use(express.static(join(__dirname, 'dist')));
```

Add SPA fallback route AFTER all API routes (after the `app.use('/api/week', weekRoutes);` line) but BEFORE `app.listen`:
```javascript
// SPA fallback — send index.html for any non-API route
app.get('*', (req, res) => {
  res.sendFile(join(__dirname, 'dist', 'index.html'));
});
```

The order matters:
1. `express.json()` — parse bodies
2. `express.static('dist')` — serve CSS, JS, images
3. `/api/*` routes — handle API requests
4. `*` catch-all — SPA fallback for client-side routing

**Step C — Add a convenience script to package.json:**

Add to `"scripts"` section:
```json
"preview": "npm run build && node server.js"
```

Do NOT modify existing `start`, `dev`, or `build` scripts.

The final server.js should look like:
```javascript
import express from 'express';
import { fileURLToPath } from 'node:url';
import { dirname, join } from 'node:path';
import { loadData, getData } from './data/store.js';
import adminRoutes from './routes/admin.js';
import memberRoutes from './routes/members.js';
import weekRoutes from './routes/week.js';

const __dirname = dirname(fileURLToPath(import.meta.url));
const app = express();
const PORT = process.env.PORT || 3000;

app.use(express.json());
app.use(express.static(join(__dirname, 'dist')));

loadData();

app.get('/api/health', (req, res) => {
  const data = getData();
  res.json({
    status: 'ok',
    timestamp: new Date().toISOString(),
    dataLoaded: data !== null,
  });
});

app.use('/api/admin', adminRoutes);
app.use('/api/members', memberRoutes);
app.use('/api/week', weekRoutes);

// SPA fallback
app.get('*', (req, res) => {
  res.sendFile(join(__dirname, 'dist', 'index.html'));
});

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

export default app;
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> static files | Browser requests for HTML/JS/CSS from dist/ |
| client -> /api/* | API requests (existing trust boundary, unchanged) |
| client -> SPA fallback | Non-API GET requests return index.html |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | express.static | mitigate | Only dist/ directory is served statically — no access to server.js, data/, or node_modules. express.static does not serve dotfiles by default. |
| T-04-02 | Denial of Service | SPA fallback catch-all | accept | Catch-all only returns a single static HTML file — minimal resource usage. Acceptable for 5-10 user scale. |
| T-04-03 | Tampering | Static file serving order | mitigate | express.static is mounted BEFORE API routes, but API routes use explicit /api/ prefix. The SPA fallback is AFTER API routes, so API requests are never intercepted by the fallback. |
</threat_model>
