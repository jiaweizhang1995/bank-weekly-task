Create `routes/members.js` (ES module, Express Router).

Import `getData` and `saveData` from `../data/store.js`. Import `requireAdmin` from `../middleware/auth.js`.

Endpoints:

1. **GET /** (no auth — per AUTH-04, member identification is by name only)
   - `const data = getData(); res.json(data.members);`
   - Returns 200 with JSON array of strings like `["李卓", "晋华", ...]`

2. **POST /** (requireAdmin middleware)
   - Body: `{ name: "string" }`
   - Validate: `name` exists, is a string, and `name.trim().length > 0`. If invalid: 400 `{ error: '请提供成员名称' }`
   - Check duplicate: `if (data.members.includes(name.trim()))` return 409 `{ error: '成员已存在' }`
   - Add: `data.members.push(name.trim()); saveData(data);`
   - Return 201 `{ message: '成员已添加', members: data.members }`

3. **DELETE /:name** (requireAdmin middleware)
   - URL param: `req.params.name` (will be URL-decoded by Express automatically)
   - Find index: `const idx = data.members.indexOf(name)`
   - If idx === -1: return 404 `{ error: '成员不存在' }`
   - Remove: `data.members.splice(idx, 1); saveData(data);`
   - Return 200 `{ message: '成员已删除', members: data.members }`

Export router as default.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Spoofing | POST /api/members | mitigate | requireAdmin middleware validates Bearer token before handler executes |
| T-02-07 | Tampering | DELETE /api/members/:name | mitigate | requireAdmin middleware required; indexOf check prevents deleting non-existent members |
| T-02-08 | Information Disclosure | GET /api/members | accept | Member names are public by design (AUTH-04: members identified by name, no auth system) — team of 5-10 people on same link |
| T-02-09 | Denial of Service | POST /api/members | accept | No rate limiting; acceptable for small trusted team. name.trim() prevents empty-string spam. |
| T-02-10 | Tampering | POST /api/members body | mitigate | Input validated: name must be non-empty string after trim(); duplicate check via includes() prevents duplicates |
</threat_model>
