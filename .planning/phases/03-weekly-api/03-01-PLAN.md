Create `routes/week.js` (ES module, Express Router).

Import `getData` and `saveData` from `../data/store.js`. Import `requireAdmin` from `../middleware/auth.js`.

Helper: define a constant for the default empty week structure used when `getData().currentWeek` is null:
```javascript
const DEFAULT_WEEK = { tasks: [], status: {}, penalty: "", deadline: "", announcement: "" };
```

When reading `currentWeek`, always use: `const week = data.currentWeek || DEFAULT_WEEK;` but when writing, ensure `data.currentWeek` is initialized first: `if (!data.currentWeek) data.currentWeek = { ...DEFAULT_WEEK };`

Endpoints:

1. **GET /** (no auth — public read per WEEK-01)
   - `const data = getData(); const week = data.currentWeek || { tasks: [], status: {}, penalty: "", deadline: "", announcement: "" }; res.json(week);`
   - Returns the full week object with all fields

2. **PUT /announcement** (requireAdmin middleware — per WEEK-02)
   - Body: `{ announcement: "string" }`
   - Validate: `announcement` key must exist in body (check `'announcement' in req.body`). If missing: 400 `{ error: '请提供公告内容' }`
   - The value can be empty string (clearing announcement is valid)
   - Initialize currentWeek if null: `const data = getData(); if (!data.currentWeek) data.currentWeek = { tasks: [], status: {}, penalty: "", deadline: "", announcement: "" };`
   - Update: `data.currentWeek.announcement = req.body.announcement; saveData(data);`
   - Return 200 `{ message: '公告已更新', announcement: data.currentWeek.announcement }`

3. **PUT /settings** (requireAdmin middleware — per WEEK-03)
   - Body: `{ deadline?: "string", penalty?: "string" }`
   - Validate: at least one of `deadline` or `penalty` must be present in body. If neither: 400 `{ error: '请提供截止时间或惩罚设置' }`
   - Initialize currentWeek if null (same as above)
   - Update only provided fields:
     ```javascript
     if ('deadline' in req.body) data.currentWeek.deadline = req.body.deadline;
     if ('penalty' in req.body) data.currentWeek.penalty = req.body.penalty;
     saveData(data);
     ```
   - Return 200 `{ message: '设置已更新', deadline: data.currentWeek.deadline, penalty: data.currentWeek.penalty }`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> GET /api/week | Public read — no auth, returns all weekly data |
| client -> PUT /api/week/announcement | Admin write — untrusted text input, requires token |
| client -> PUT /api/week/settings | Admin write — untrusted deadline/penalty input, requires token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | PUT /api/week/announcement | mitigate | requireAdmin middleware validates Bearer token before handler executes |
| T-03-02 | Spoofing | PUT /api/week/settings | mitigate | requireAdmin middleware validates Bearer token before handler executes |
| T-03-03 | Information Disclosure | GET /api/week | accept | Week data is public by design — entire team sees same tasks, same statuses. Team of 5-10 on same link. |
| T-03-04 | Tampering | PUT /api/week/announcement body | mitigate | Validate `announcement` key exists in body; value stored as-is (plain text, rendered as text in frontend, no XSS vector since React escapes by default) |
| T-03-05 | Tampering | PUT /api/week/settings body | mitigate | Validate at least one field present; deadline/penalty stored as strings matching frontend expectations |
| T-03-06 | Denial of Service | PUT endpoints | accept | No rate limiting; acceptable for small trusted team (5-10 users) |
</threat_model>
