Create `data/store.js` as an ES module exporting three functions:

1. `loadData()` — Synchronously reads `data/data.json` using `fs.readFileSync`. If the file or directory does not exist, creates the `data/` directory with `fs.mkdirSync(path, { recursive: true })`, writes the defaultData object as JSON with 2-space indent, and returns it. Stores the parsed object in a module-level `let cache = null` variable.

2. `saveData(newData)` — Writes `newData` to `data/data.json` using `fs.writeFileSync` with 2-space indent. Updates the module-level cache. Returns nothing.

3. `getData()` — Returns the current cache. If cache is null, calls loadData() first. This is for request handlers to get data without repeated disk reads.

The defaultData constant:
```js
const defaultData = {
  adminPin: "8888",
  members: ["李卓", "晋华", "珊珊", "晓梅", "张伟"],
  tasks: [],
  currentWeek: null,
};
```

Use `import { readFileSync, writeFileSync, mkdirSync, existsSync } from 'node:fs'` and `import { join, dirname } from 'node:path'` and `import { fileURLToPath } from 'node:url'`.

Compute the data file path relative to the module: `const __dirname = dirname(fileURLToPath(import.meta.url))` then `const DATA_PATH = join(__dirname, 'data.json')`.

**Step A — Install Express:**

Run `npm install express` in the project root. This adds express to package.json dependencies and updates package-lock.json.

**Step B — Create server.js:**

Create `server.js` in the project root as an ES module:

```js
import express from 'express';
import { loadData, getData } from './data/store.js';

const app = express();
const PORT = process.env.PORT || 3000;

// Parse JSON bodies
app.use(express.json());

// Initialize data on startup
loadData();

// Health check
app.get('/api/health', (req, res) => {
  const data = getData();
  res.json({
    status: 'ok',
    timestamp: new Date().toISOString(),
    dataLoaded: data !== null,
  });
});

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

export default app;
```

Key details:
- Port defaults to 3000 but respects `PORT` env var (for deployment flexibility)
- `loadData()` called at module level before listen — ensures data.json exists
- Health check at `/api/health` returns JSON with `status: "ok"`, current timestamp, and `dataLoaded: true`
- `export default app` allows future test imports
- `express.json()` middleware ready for POST/PUT bodies in later phases

**Step C — Add start script to package.json:**

Add to the `"scripts"` section: `"start": "node server.js"`

Do NOT modify existing "dev" or "build" scripts.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> Express API | HTTP requests from browser to server (untrusted input) |
| Express -> filesystem | Server reads/writes data.json (trusted internal) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | express.json() body parsing | mitigate | Express built-in JSON parser rejects malformed bodies with 400. Payload size will be limited in Phase 2 when write endpoints are added. |
| T-01-02 | Denial of Service | /api/health endpoint | accept | Health check is read-only, returns tiny JSON. No auth needed. Acceptable for 5-10 user scale. |
| T-01-03 | Information Disclosure | /api/health response | mitigate | Health check returns only status, timestamp, and dataLoaded boolean. No internal paths, versions, or stack traces exposed. |
| T-01-04 | Tampering | data.json on disk | accept | Server runs on private ECS. File permissions are OS-level concern. Acceptable for single-server deployment. |
</threat_model>
